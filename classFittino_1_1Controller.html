Detailed Description

Singleton class for controlling the execution flow of Fittino.

The instance of the Controller class is the first object that is created at the beginning of
the execution of Fittino. Depending on the user configuration the controller creates
instances of further objects (either directly or with the help of a factory) and advises
them to perform the specified tasks, hereby controlling the program's overall execution
flow.

The controller adresses its purpose in three distinct phases: At first, the controller
initializes Fittino by letting an input file interpreter read in user specified options.
After that further objects are created and put into action according to the user's
configuration. At the end, the controller provides the controlled termination of Fittino.
